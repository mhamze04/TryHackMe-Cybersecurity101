# 🛡️ Introduction to Cyber Security

Ce module pose les bases théoriques et éthiques de la cybersécurité, en explorant les deux mondes : l'Offensif et le Défensif.

## ⚔️ Offensive Security (Red Team)
L'approche offensive consiste à tester la sécurité en essayant de s'introduire dans les systèmes (Pentesting).
- **Objectif** : Identifier les failles avant qu'un véritable attaquant ne les exploite.
- **Mentalité** : "Pour protéger un système, il faut savoir comment il peut être attaqué."

## 🛡️ Defensive Security (Blue Team)
L'approche défensive se concentre sur la protection des actifs et la détection des intrusions.
- **Objectif** : Analyser le trafic, durcir les configurations (Hardening) et répondre aux incidents.
- **Outils clés** : SIEM, IPS/IDS, Analyse de Logs.

## 🔍 Search Skills (OSINT & Research)
Un expert en cyber doit savoir trouver l'information là où elle se cache.
- **Google Dorking** : Utilisation d'opérateurs avancés pour trouver des fichiers sensibles ou des serveurs exposés.
    - `site:target.com filetype:pdf` (Trouver tous les PDF d'un domaine précis).
    - `intitle:"index of" /admin` (Rechercher des répertoires d'administration ouverts).
- **Documentation** : Savoir lire une CVE (Common Vulnerabilities and Exposures) pour comprendre une faille.

## 📝 Ce que j'ai retenu
La cybersécurité est un équilibre permanent entre l'attaque et la défense. J'ai appris que la phase de recherche d'information (Reconnaissance) est souvent la plus importante d'une opération : mieux on connaît sa cible ou son infrastructure, plus on est efficace.
